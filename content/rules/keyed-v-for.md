# Keyed `v-for`

`key` with `v-for` is always required on components, in order to maintain internal component state down the subtree.

## Rule for AI agents

```
- ALWAYS use key in v-for loops
```

## Eslint rule

- https://eslint.vuejs.org/rules/require-v-for-key.html
